# HAP: Архитектура проекта и дизайн

**Цель:** Описать организацию репозитория, взаимосвязи между компонентами и контракты API для ключевых функций. HAP — это в первую очередь теоретический и исследовательский проект, поэтому кодовая база сосредоточена на анализе данных, моделировании и управлении доказательствами.

## 2. Ключевые компоненты и их взаимодействие

1.  **`core/`** — Единый источник истины. Все остальные компоненты должны быть согласованы с информацией в этих файлах.
2.  **`data/parameters.yaml`** — Машиночитаемое представление таблицы из `PARAMETERS.md`. Автоматически генерируется/проверяется скриптом `scripts/update_parameters.py`.
3.  **`models/`** — Jupyter ноутбуки и скрипты для теоретического моделирования (например, минимальная модель связи концентрация-поведение) и анализа данных (например, статистическая обработка поведенческих экспериментов).
4.  **`scripts/verify_evidence.py`** — Критический скрипт поддержания целостности. Парсит `EVIDENCE.md`, проверяет каждую ссылку через PubMed/Crossref API и генерирует отчёт о "битых" или непроверенных ссылках.
5.  **`JOURNAL.md`** — Фиксирует все изменения. Любое изменение в `core/` файлах, значимое обновление данных или скриптов должно сопровождаться записью в `JOURNAL.md`.
