## РЕЗЮМЕ ДЛЯ АВТОРА

**Проект:** Organismal Aging — самообучающийся 4D-симулятор организма, интегрирующий центриолярную теорию старения (CDATA), многокомпонентную архитектуру (MCAOA) и теорию Ze-конфликтов в единую вычислительную модель.

**Общая оценка:** Проект заполняет реально существующую научную нишу. PubMed-поиск по запросу `multi-scale computational model aging whole body simulation` (2026-06-21) — **ноль результатов**. Ни один из существующих симуляторов (McAuley — метаболизм, Kirkwood — disposable soma, Farrell — agent-based, Passos — senescence) не объединяет три масштабных уровня (субклеточный → молекулярный → тканевый) в единую механистическую модель с физической валидацией через робота.

**Вердикт:** Принять с серьёзными доработками. Теоретический фундамент заложен верно, код работает, но доказательная база требует существенного усиления по 8 позициям.

## §1. СИЛЬНЫЕ СТОРОНЫ

### 1.1. Новизна архитектуры
Трёхуровневая архитектура (Центриоль → 5 счётчиков MCAOA → 8 тканей + Ze-конфликты) не имеет аналогов. PubMed-поиск 2026-06-21 по комбинациям `centriole AND aging AND computational model`, `multi-scale aging simulation` — существующие модели либо одноуровневые (метаболизм, эпигенетика), либо чисто статистические (Horvath clock, GrimAge), либо концептуальные (López-Otín hallmarks).

**Подтверждение уникальности:**
- McAuley et al. (2017) — цельноорганизменная модель метаболизма, но без клеточного/субклеточного уровня
- Kirkwood (1977) disposable soma — математическая модель без конкретных молекулярных механизмов
- Farrell et al. (2023) — agent-based модель тканевого старения, без субклеточных счётчиков
- López-Otín et al. (2023, PMID 36599349) — 12 hallmarks, описательная классификация, не вычислительная

**Organismal Aging — первый симулятор, объединяющий конкретный субклеточный механизм (центриоль) с молекулярными счётчиками и тканевыми конфликтами.**

### 1.2. Интеграция с физической валидацией
ARGUS-LP — не просто симулятор, а роботизированная платформа для тестирования гипотез симулятора на физических объектах (клетки, INFOGEST, органоиды). Это концептуально новый подход: self-learning loop, где модель генерирует гипотезу, робот тестирует её, результат возвращается через байесовское обновление.

**Аналог:** Digital twins в инженерии (Grieves 2017), но в биологии старения — первый случай.

### 1.3. Строгость формализма
Формулы S_centriole(t), L_tissue, Z_conflict(i,j) имеют чёткие математические определения. Каждый параметр аннотирован источником (provenance tracking: PMID/DOI/Estimate/Postulate). 44 теста Rust подтверждают корректность реализации.

### 1.4. Доказательная база центриолярной теории
Центриолярная теория старения (CDATA) имеет прямую PubMed-поддержку:
- **PMID 36583780** (Tqemaladze 2023) — обзорная статья, формулирующая гипотезу
- **PMID 17255513** (Yamashita 2007, *Science*) — асимметричное наследование центросомы в стволовых клетках Drosophila
- **PMID 15886028** (Tqemaladze 2005) — ранняя работа о роли центриолей в морфогенезе
- **PMID 19829375** (Wang 2009, *Nature*) — асимметричное наследование центросомы поддерживает нейральные прогениторы в неокортексе

## §2. КРИТИЧЕСКИЕ ЗАМЕЧАНИЯ (требуют обязательного ответа)

### 2.1. Энтропийный постулат не подтверждён экспериментально 🔴

**Проблема:** Утверждение «Центриоль — единственная структура без самообновления → накопитель энтропии → первопричина старения» является постулатом, не подтверждённым прямыми измерениями.

**Что есть в литературе:**
- Энтропия как маркер старения подтверждена на других системах: мышечные волокна (PMID 41724675, Hong 2026), метилирование ДНК (PMID 40096548, Chan 2025; PMID 36797759, Vaidya 2023), хроматин (PMID 23058597, Pantic 2012), C. elegans (PMID 19808878, Shamir 2009)
- Диссипативное масштабирование старения многоклеточных (PMID 38367762, Kriete 2024)
- Клеточная «информационная энтропия» мутаций ДНК (PMID 37659107, Yu 2023)
- **Но ни одно из этих исследований не измеряло энтропию центриоли напрямую.**

**Требование рецензента:**
1. Чётко обозначить гипотезу центриолярной энтропии как **постулат, ожидающий экспериментальной проверки** (ARGUS Phase B)
2. Добавить ссылки на косвенные подтверждения энтропийного старения из других систем (PMID 41724675, 40096548, 36797759, 38367762)
3. Сформулировать конкретный фальсифицируемый тест: «Если polyGlu на материнской центриоли НЕ коррелирует с числом делений (r < 0.3 при N≥100) — гипотеза опровергнута»

### 2.2. Калибровка L_tissue через Frailty Index — логический круг 🔴

**Проблема:** Утверждается, что L_tissue калибруется через FI (Rockwood 2005, PMID 16129869) по формуле L = FI/0.7. Однако в симуляторе используется обратная формула FI = 0.7 × L_max, где L_max вычисляется из модели. Возникает кажущийся логический круг: модель предсказывает L, которое преобразуется в FI, которое сравнивается с литературными FI, которые использовались для калибровки L.

**Реальность:** Круг не замкнут, поскольку:
- L_tissue вычисляется из первых принципов (S_centriole + счётчики)
- FI = 0.7 × L_max — это операционализация для сравнения с клиническими данными
- Параметры модели (η, α, β, w_i) НЕ подгонялись под FI — они взяты из независимых источников

**Требование рецензента:**
1. Явно прописать в CONCEPT.md и THEORY.md: «Параметры модели взяты из независимых биологических источников (Хейфлик, DunedinPACE, turnover times); FI используется только как операциональный мост для сравнения с клиническими данными, а не для калибровки параметров»
2. Добавить ссылку на Searle 2008 (PMID 18671847) — стандартный протокол FI
3. Добавить ссылку на Mitnitski 2002 (PMID 12456714) — связь FI со смертностью

### 2.3. Отсутствие независимой валидации 🔴

**Проблема:** Симулятор выдаёт правдоподобные результаты (смерть в 101 год, болезни в 40-60 лет), но это НЕ валидация — это внутренняя согласованность модели с её же параметрами.

**Требование рецензента:**
1. Предрегистрировать протокол валидации на OSF до начала калибровки на реальных данных (GTEx, UK Biobank)
2. Указать конкретные метрики успеха: «Модель считается валидированной, если предсказанный возраст смерти для UK Biobank когорты имеет среднюю абсолютную ошибку (MAE) < 8 лет»
3. Добавить сравнительный анализ с существующими предикторами (GrimAge, PhenoAge, DunedinPACE)

## §3. СУЩЕСТВЕННЫЕ ЗАМЕЧАНИЯ (необходимо адресовать)

### 3.1. Неравномерность доказательной базы по счётчикам 🟡

| Счётчик | Уровень доказательности | Проблема |
|---|---|---|
| #1 Центриоль | Средний | Теория + Drosophila; человек не подтверждён |
| #2 Теломеры | Высокий | Хейфлик 1965 — классика; сотни подтверждений |
| #3 Митохондрии | Средний | Много косвенных данных; нет прямой связи с лимитом делений |
| #4 Эпигенетика | Высокий | Эпигенетические часы хорошо валидированы |
| #5 Протеостаз | Низкий | VEXAS — патология, не физиологическое старение; PPIA — одна работа (Maneix 2024) |

**Требование:** Для каждого счётчика указать уровень достоверности (confidence score) и конкретный план повышения.

### 3.2. Ze-конфликты — гипотеза без эмпирической базы 🟡

**Проблема:** Z_conflict(i,j,t) = |τ_i·dL_i/dt − τ_j·dL_j/dt| · C_ij — элегантная формула, но:
- C_ij (матрица связности тканей) полностью неизвестна
- Ни одного прямого эксперимента, подтверждающего что рассогласование скоростей старения тканей вызывает болезнь
- Ze-Hierarchy (боты) — интересная аналогия, но не доказательство для биологии

**Требование:**
1. Предложить хотя бы один эксперимент для измерения C_ij (например, ко-культуры старых и молодых клеток разных тканей)
2. Привести косвенные подтверждения из литературы: сердечно-сосудистые заболевания как конфликт эндотелия и кардиомиоцитов → ссылка на Lakatta (PMID 12586292)
3. Явно обозначить Z_conflict как гипотезу, а не установленный механизм

### 3.3. INFOGEST-интеграция не завершена 🟡

Модель пищеварения (DigestionResult) вычисляет ROS и протеостазные множители, но они НЕ подключены к основному циклу симуляции (Organism::step). Диеты не влияют на продолжительность жизни в текущей версии.

**Требование:** Реализовать `apply_diet()` в Organism.

### 3.4. Мёртвые модули 🟡

`microbiome/`, `learning/`, `spatial/` — содержат только TODO-комментарии. Для v0.1 это приемлемо, но должно быть явно отражено в STATE.md и ROADMAP.

## §4. МЕТОДОЛОГИЧЕСКИЕ ЗАМЕЧАНИЯ

### 4.1. Проблема размерности времени в счётчиках

Формула D_i(n,t) = α_i·(n/n*) + β_i·(t/τ*) чувствительна к выбору τ*. Для митохондрий τ* варьирует от 3 дней (mtDNA turnover) до 30 дней (митофагия), что меняет предсказания в 10 раз.

**Требование:** Зафиксировать используемые τ* с обоснованием и цитированием.

### 4.2. Сигмоидальная функция — произвольный выбор

f_i(x) = 1/(1+exp(-k(x−x_crit))) с k=8 — произвольный выбор. Почему не k=5? Не k=12? Форма сигмоиды сильно влияет на возраст начала болезней.

**Требование:** Провести анализ чувствительности по k и x_crit. Обосновать выбор канонических значений.

### 4.3. Отсутствие анализа неопределённости

Модель имеет >30 свободных параметров (η, α_i, β_i, k_i, x_crit_i, w_i, τ_i, C_ij...). При таком числе параметров риск переобучения высок.

**Требование:**
1. Провести анализ чувствительности (Sobol) для выявления доминирующих параметров
2. Оценить доверительные интервалы для предсказаний (бутстрэп / MCMC)

## §5. НЕЗАВИСИМАЯ ПРОВЕРКА ФОРМУЛ

Рецензент проверил ключевые формулы на внутреннюю согласованность:

### 5.1. Центриолярная энтропия
```
S(t) = S₀ + ∫₀ᵗ [η_div·n(τ)/n* + η_time·τ/τ* + η_ROS·ROS(τ) + η_rep·ε_rep(τ)] dτ
```
✅ Размерностно корректна: все члены под интегралом — скорости (1/время × безразмерные множители)
✅ Интеграл даёт безразмерную величину
⚠️ Член η_ROS·ROS(τ) не включает нормировку — ROS(τ) должен быть безразмерным [0,1]

### 5.2. Агрегатор L_tissue
```
L_tissue = w_centriole·S + Σᵢ₌₂⁵ w_i·f_i(D_i)
```
✅ Σ весов = 1.0 (центриоль + 4 счётчика)
⚠️ w_centriole вычисляется как остаток (1 − Σw_i), а не из независимых данных

### 5.3. Ze-конфликт
```
Z_conflict(i,j) = |τ_i·dL_i/dt − τ_j·dL_j/dt| · C_ij
```
✅ Размерностно: [время] × [1/время] × [безразмерная] = безразмерная
⚠️ dL/dt вычисляется через скользящее среднее с α=0.1 — произвольный выбор параметра сглаживания

### 5.4. Кодовая проверка
