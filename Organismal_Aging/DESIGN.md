## §2. Ключевые абстракции (Rust traits)

```rust
/// Счётчик старения (Уровень #2)
trait AgingCounter {
    fn name(&self) -> &str;
    fn damage(&self, divisions: f64, time: f64, ros: f64) -> f64;
    fn critical_threshold(&self, tissue: &Tissue) -> f64;
    fn default_params() -> Self;
}

/// Ткань (Уровень #3)
trait TissueLike {
    fn name(&self) -> &str;
    fn renewal_period(&self, species: &Species) -> f64;
    fn counter_weights(&self) -> [f64; 5];
    fn critical_burden(&self) -> f64;
    fn spatial_position(&self) -> [f64; 3];
}

/// Вид
trait SpeciesLike {
    fn name(&self) -> &str;
    fn max_lifespan(&self) -> f64;
    fn tissues(&self) -> Vec<Box<dyn TissueLike>>;
    fn has_centrioles(&self) -> bool;
    fn counters_active(&self) -> Vec<usize>;
}
```

## §3. Поток данных

```
Конфигурация (YAML/JSON)
  │
  ▼
SpeciesConfig → Organism::new(species)
  │
  ▼
Цикл симуляции (t = 0 → T_max, шаг Δt):
  │
  ├─→ centriole::update(dt)         ← S_centriole(t)
  ├─→ counters::update(dt)          ← Dᵢ(t) для всех счётчиков
  ├─→ tissue::update(dt)            ← L_tissue(t), Z_conflict(t)
  ├─→ organism::check_events(dt)    ← Болезни, травмы, смерть
  └─→ learning::maybe_update(dt)    ← Байесовское обновление
  │
  ▼
Результат: AgingCurve { t, S_centriole, L_tissues, Z_conflicts, FI, events }
```

## §6. Масштабирование и производительность

### 6.1 Вычислительные требования

```
Полная симуляция (человек, 120 лет, Δt=1 час):
  120 × 365 × 24 = 1,051,200 шагов
  На шаг: centriole + 4 counters + 8 tissues + Ze(8×8) + organism
  ~200 операций/шаг
  Итого: ~210 млн операций

Текущая производительность (single-thread, release):
  120 лет × Δt=1 год = 120 шагов < 1 мс
  Экстраполяция: 1 млн шагов ~ 8 секунд

С ансамблем Монте-Карло (N=1000):
  ~8000 секунд = ~2.2 часа (single-thread)
```

### 6.2 Стратегия оптимизации

| Уровень | Метод | Ускорение |
|---|---|---|
| **Алгоритмический** | Адаптивный Δt: 1 час для быстрых тканей, 1 день для медленных | 10-100× |
| **Параллелизм** | Rayon: параллельное обновление тканей + ансамблей | 4-8× |
| **GPU** | CUDA для Монте-Карло ансамблей (1000+ симуляций) | 50-100× |
| **Кэширование** | LUT для сигмоиды, RingBuffer для истории | 20-30% |
| **Редукция модели** | PCA/Sobol для сокращения размерности θ | 2-5× |

### 6.3 Режимы работы

| Режим | Описание | Производительность |
|---|---|---|
| **Интерактивный** | Δt=1 год, 1 симуляция | < 1 мс |
| **Исследовательский** | Δt=1 день, 1 симуляция | ~1 сек |
| **Ансамблевый** | Δt=1 день, 1000 симуляций (GPU) | ~10 сек |
| **Полный** | Δt=1 час, 1 симуляция | ~8 сек |
| **Калибровочный** | MCMC, 10,000 итераций, ансамбль | ~1 час (GPU) |
